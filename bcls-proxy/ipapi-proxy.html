ipapi-proxy

Version: 0.1.0

Author: Robert Crooks, Director of Learning Services, Brightcove, Inc.

Description: Proxies Brightcove Ingest Profiles API requests, getting an access token, making the call, and returning the response to an iframe on the client page. This proxy will only handle requests originating from a Brightcove domain or localhost. For a guide to building a proxy like this one, see http://docs.brightcove.com/en/video-cloud/oauth-api/guides/quick-start.html

This is not a production app. By design, this proxy will reject any request that does not originate from a brightcove.com domain or localhost. You can use this proxy to test your app if you have a local web server (such as MAMP or XAMPP).

Note: you have probably reached this page because you submitted a request to the proxy that contained no data. See the requirements for using the proxy below.

Change log

  • 0.1.0: initial release 2014-10-07

Requirements:

POST your request to:

Server URL Description
solutions.brightcove.com:8006 CMS API

Target an iframe on your page to display the response (unless using AJAX)

Required fields for the body:

These should be submitted with a Content-Type:application/x-www-form-urlencoded header, and the url should be encodeURIComponent encoded.

Data field Required Description
client_id yes[1] get from the Brightcove OAuth UI in Studio[1]
client_secret yes[1] get from the Brightcove OAuth UI in Studio[1]
requestType no GET (default) | POST | PUT | PATCH | DELETE
url yes full url for the API request
Notes

[1] Alternatively, you can obtain a client id and secret from the OAuth API directly - see Getting Client Credentials using CURL or Getting Client Credentials using Postman
